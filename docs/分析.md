有查看过。boxes下的文件是以csv形式存储，其中列元素有label,x,y,z,w,h,l,row,pitch,yaw,是单株的数据。colored_maps、depth_images、images下都是图片.

`boxes` 里的 CSV 文件刚好提供了我们需要的核心信息：三维空间坐标 (x, y, z)、物理尺寸 (w, h, l) 以及三维姿态角。

`roll`（翻滚角）📐，它和 pitch（俯仰角）、yaw（偏航角）共同构成了 6DoF 里的旋转姿态。

网络需要回归一个包含 22 维特征的向量，其中包括中心点在图像坐标系下的偏移，以及 8 个顶点在图像坐标系下的坐标偏移 。这总共是 9 个关键点（Keypoints）。

### 相机内参矩阵：

 u, v = 400.32 * X / Z + 400, 400.32 * Y / Z + 300

根据针孔相机模型公式，可以推出：

焦距：fx=400.32，fy=400.32

主点坐标（光学中心）：cx=400,cy=300.

图像分辨率是800*600

`X, Y, Z = corners_3d[i][0], -corners_3d[i][1], -corners_3d[i`][2]

将 3D 点投影到 2D 图像之前，必须把 Y 和 Z 的符号反转。

网络最终要预测一个包含二维关键点投影信息的高维回归变量。在读取数据时，也要把九个二维关键点的坐标计算出来并返回。这样Darknet+Transformer网络训练时才知道要往什么2D坐标去回归。

求网格列索引 `grid_x` 和相对偏移量 `offset_x` 的数学逻辑是：

grid_x = int(u / 40)
offset_x = (u - grid_x * 40) / 40.0

引入变量 `W`（图片宽度）和 `S`（网格列数）后

grid_width = W / S
grid_x = int(u / grid_width)
offset_x = (u / grid_width) - grid_x

都是水平方向

计算中心点所在的网格索引和相对偏移量的完整代码

```python
# 假设 W, H 为图像宽高，S 为网格划分数量
grid_w, grid_h = W / S, H / S

# 中心点所在网格的行列索引
grid_x, grid_y = int(u / grid_w), int(v / grid_h)

# 中心点相对于网格左上角的偏移量
offset_x = (u / grid_w) - grid_x
offset_y = (v / grid_h) - grid_y
```

通过计算顶点相对于草莓中心点的偏移量，网络实际上是在学习草莓的几何拓扑结构和3D空间姿态。

其余8个顶点的u,v偏移量计算如下述代码：

```python
# 提取中心点坐标
center_u, center_v = pts_2d[8]

# 计算 8 个顶点的偏移量，并交替存入一个列表中 [du0, dv0, du1, dv1, ...]
vertex_offsets = []
for i in range(8):
    delta_u = pts_2d[i][0] - center_u
    delta_v = pts_2d[i][1] - center_v
    vertex_offsets.extend([delta_u, delta_v])
```

最后拼接成22维向量的代码如下所示：

```python
# 1. 中心点偏移 (2维)
center_offsets = [offset_x, offset_y]

# 2. 顶点偏移 (16维) - 刚才已经算好了保存在 vertex_offsets 列表里
# vertex_offsets = [du0, dv0, du1, dv1, ...]

# 3. 三维物理尺寸 (3维)
size_3d_list = [w, h, l]

# 4. 目标置信度 (1维) - 真实存在的标签置信度设为 1.0
confidence = [1.0]

# 像接火车一样把所有列表首尾相连，变成长度为 22 的一维列表 🧩
target_vector = center_offsets + vertex_offsets + size_3d_list + confidence

# 最后一步：统一转换为 PyTorch 需要的 float32 张量
target_tensor = torch.tensor(target_vector, dtype=torch.float32)
```

这样网络在训练时直接学习这个22维向量就好了。
