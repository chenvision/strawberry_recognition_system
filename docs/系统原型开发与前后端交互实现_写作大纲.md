# 第X章 系统原型开发与前后端交互实现

## 4.1 系统开发目标与总体架构
### 4.1.1 原型系统开发目标
*   **功能性目标：** 实现从图像采集、模型推理到结果反馈的全链路集成。
*   **实时性目标：** 优化前后端传输链路，确保姿态估计结果的低延迟反馈。
*   **可视化目标：** 将抽象的 6D 姿态数据转化为直观的 3D 几何模型叠加层。

### 4.1.2 技术栈选型
*   **前端框架：** Vue.js 3 + Axios（负责 UI 逻辑与状态管理）。
*   **后端引擎：** Python FastAPI / Flask（负责深度学习模型调度与图像处理）。
*   **通信协议：** 基于 HTTP 的 RESTful API（或 WebSocket，视实现而定）。

---

## 4.2 后端推理服务的构建
### 4.2.1 推理引擎集成
*   基于 PyTorch 的模型权重热加载机制。
*   推理流水线设计：图像预处理（Resize, Normalization） -> 模型推理 -> 后处理（NMS, 姿态解码）。

### 4.2.2 推理接口 API 设计
*   **Endpoint:** `/predict`
*   **输入规范：** Base64 编码的图像数据或 Multipart 文件流。
*   **输出规范：** 结构化 JSON（包含 3D 框顶点、旋转矩阵 $R$、平移向量 $T$、长宽高尺寸及置信度）。

---

## 4.3 前端交互界面与逻辑实现
### 4.3.1 多源图像采集模块
*   **摄像头实时流：** 基于浏览器 MediaDevices API 的实时帧捕获技术。
*   **本地文件上传：** 离线测试数据的批量读取与预览。

### 4.3.2 异步状态管理机制
*   基于 `useInferenceLoop.js` 的循环调度策略。
*   “请求-响应-渲染”周期的状态机切换，确保界面的非阻塞交互。

## 4.5 姿态与尺寸的实时三维可视化 (重点)
### 4.5.1 基于相机内参的重投影技术
*   **投影模型：** 阐述从 3D 空间坐标到 2D 像素平面的透视投影公式。
*   **顶点计算：** 描述如何根据中心点、姿态矩阵和长宽高计算出 3D 边框的 8 个顶点坐标。

### 4.5.2 3D 几何模型渲染
*   **立体框绘制：** 使用 Canvas/SVG 绘制具有透视感的 12 条棱边。
*   **坐标轴可视化：** 在草莓质心处绘制 RGB 三色坐标轴，直观展示 Pitch/Yaw/Roll 角度。

### 4.5.3 物理尺寸的实时标注
*   如何将估计的毫米级长宽高数值动态叠加在 3D 框上方。
*   可视化反馈与真实物体尺寸的一致性说明。
