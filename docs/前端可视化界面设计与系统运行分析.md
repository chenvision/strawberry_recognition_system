# 第X章 前端可视化界面设计与系统运行分析

## X.1 前端可视化界面设计概述
本系统的用户界面（UI）采用响应式设计理念，基于 Vue.js 3 框架构建。界面设计不仅追求美观性，更核心的目标是实现算法输出结果的**具象化、交互化与透明化**。界面整体分为侧边功能切换栏、中心视觉呈现区、以及右侧实时数据看板三大板块，通过异步状态机管理图像流与推理结果的同步。

---

## X.2 核心功能模块详细设计

### X.2.1 实时监测模块 (Real-time Monitoring)
实时监测模块是面向实际应用场景（如自动化采摘作业）的核心功能，其主要目标是实现草莓 3D 位姿的高动态响应。

*   **实现机制：** 该模块通过前端 `useCamera.js` 钩子调用浏览器的 MediaDevices API 获取摄像头视频流，并由 `useInferenceLoop.js` 驱动一个异步生产-消费循环。前端将视频帧实时抽取并发送至后端 `/api/predict` 接口，在获取模型返回的姿态参数后，立即利用 HTML5 Canvas 进行覆盖渲染。
*   **功能特点：** 
    *   **低延迟交互：** 采用非阻塞式网络请求，确保界面在等待推理结果时仍能保持流畅的视频预览。
    *   **动态位姿跟踪：** 能够随着草莓或相机的移动，实时更新 3D 边框的投影形态，直观展示草莓在三维空间中的运动轨迹与姿态倾角。
    *   **自动重连机制：** 针对复杂的硬件环境，设计了指数退避算法的摄像头自动重连逻辑，增强了系统的鲁棒性。

### X.2.2 图片分析模块 (Image Analysis)
图片分析模块主要用于离线科研验证与算法精度回测，支持对高质量静态图像进行深度解析。

*   **实现机制：** 用户可通过点击或拖拽方式上传本地 `.jpg` 或 `.png` 格式图像。前端将图像封装为 `FormData` 后调用后端的 `analyze_image` 接口。与实时模式不同，该模式下后端会执行更高精度的后处理算法，并返回包含 3D 渲染效果的 Base64 底图及结构化位姿数据。
*   **功能特点：**
    *   **精准尺寸测算：** 该模式侧重于展示草莓的**物理参数**，包括长、宽、高的毫米级（mm）估算值，并将其动态标注在结果看板中。
    *   **多目标协同展示：** 系统能够在一个画面内同时解析并渲染多个草莓目标的 3D 位姿，通过点击不同的识别框，右侧看板会实时切换显示对应目标的 3D 空间坐标（X, Y, Z）。

### X.2.3 标准参考集回放模块 (Standard Reference Replay Module)
该模块是系统的“真值验证中心”，主要用于验证可视化引擎的准确性，并作为评估模型表现的“黄金基准”。

*   **实现机制：** 模块直接对接 Straw6D 数据集中的原始标注数据。当用户在“演示图库”中选择特定样本时，后端读取对应的 CSV 标注文件（包含旋转矩阵 $R$、平移向量 $T$ 的真值），前端则利用预设的相机内参矩阵 $K$ 执行逆投影运算。
*   **功能特点：**
    *   **渲染逻辑校验：** 它是证明“可视化代码”正确性的唯一手段。若真值数据生成的 3D 框能够完美贴合图像中的草莓边缘，则证明系统的相机几何模型与重投影算法是严丝合缝的。
    *   **量化对比基准：** 在论文的对比实验中，该模块提供的 Ground Truth (GT) 效果图常作为基准图，与算法生成的预测图并列展示，以直观说明 6D 姿态估计的几何误差。

## X.3 系统运行分析与性能评估

### X.3.1 可视化渲染流程分析
系统通过 `strawberryParsers.js` 模块，将算法输出的抽象张量（Tensor）映射为几何顶点。渲染逻辑遵循以下流水线：
1.  **解析：** 将旋转向量（Euler/Quaternion）转换为旋转矩阵 $R$。
2.  **变换：** 计算草莓中心点 $T$ 与 8 个顶点的三维坐标。
3.  **投影：** 应用相机内参 $K$ 将 3D 点投影至 2D 像素平面。
4.  **绘制：** 在 Canvas 层绘制 12 条具有透视感的边线及 RGB 三色局部坐标轴。

### X.3.2 响应性能分析
经过测试，在本地环境下，“采集-上传-推理-返回-渲染”的完整生命周期耗时约为 120ms - 200ms（视 GPU 负载而定）。前端通过 Vue 的响应式系统监听 `displayTargets` 数组的变化，并在每一帧渲染前执行 `ctx.clearRect()` 以清除上一帧残影，确保了可视化叠加层与背景画面的高贴合度。
